1.4 QM1 – Session 4 (Data)

Data types

Many data types are available in R. The most basic data type logical has been introduced already implicitly. It takes either TRUE or FALSE. Also T or F as global variables can be used. It is not advised to use those, as they can be over written. Furthermore, it is strongly discouraged to assign objects to T or to F.

myfirsttype <- c(TRUE, FALSE)  # assigns a vector of TRUE and FALSE to 'myfirsttype'
str(myfirsttype)  # returns the structure of 'myfirsttype', mind the 'logi'
##  logi [1:2] TRUE FALSE

class(myfirsttype)  # the object has class "logical"
## [1] "logical"

typeof(myfirsttype)  # the object is of type "logical"
## [1] "logical"

if (T) cat("Woo Hoo!") else cat("D'oh!")  # Do NOT use 'T'!
## Woo Hoo!

T <- FALSE  # Do NOT assign objects to 'T'!!
if (T) cat("Woo Hoo!") else cat("D'oh!")  # Do NOT use 'T'! You see why not!!!
## D'oh!

rm(T)  # Remove the object 'T' right away!!!!
if (T) cat("Woo Hoo!") else cat("D'oh!")  # Although it works: Do NOT use 'T'!!!!!
## Woo Hoo!

The most common type for real numbers is double.

Therefore, a short introduction to the theory of number representation is given. In mathematics, every x ∈ ℝ can be decomposed as

x = σ ∑ k=1∞a k2−k 2e

whereσ ∈{±1}is the sign of the number,
ak ∈{0, 1}is the digit
ande ∈ ℤis the exponent.

This representation is not unique, since e. g. 

1 = 1 ∑ k=1∞2−k 20 = 1 ∑ k=112−k 21

A more familiar representation will be

x = σ ∑ k=1∞a k10−k 10e

which is isomorph to the decimal system.

Because today’s computers only know applied voltage or no voltage, computers work with a cutted of version of the binary representation, a floating point system F(2,M,emin,emax) ⊂ ℚ, where the floating point number is a function of the system basis (2 for the binary system), the number of significant digits M ∈ ℕ and the limits of the exponent emin ≤ 0 ≤ emax. x ∈ F is represented by

x = σ ∑ k=1Ma k2−k 2e

a1 = 1 in nearly all aplications, the so called hidden bit, and, thus, is usally not saved.

Named sequences and how to access elements thereof

Elements of vectors can be named and then called by their names

myVector <- 1:3  # "myVector" gets "1:3"
myVectorNames <- c("one", "two", "three")  # assignes names to "myVectorNames"
names(myVector)  # returns "NULL", i.e. there are no names
## NULL

names(myVector) <- myVectorNames  # the names of the elements get "myVectorNames"
names(myVector)  # returns the names of the elements, which are not "NULL" any more
## [1] "one"   "two"   "three"

myVector  # the elements are named
##   one   two three  
##     1     2     3

myVector[2]  # the elements can be accessed by their position ...
## two  
##   2

myVector["two"]  # or the elements can be accessed by their name
## two  
##   2
