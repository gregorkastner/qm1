In plain R you only have the console to input commands. This is one (endless) line, in which you can write meaningful texts, e. g. commands, calculations, et cetera. By pressing Enter (or Return), these lines of text get interpreted and executed. Ris ready for input, when > is displayed. For example typing 1 + 2 and pressing Return, R returns in the line below the input [1] 3. [1] indicates that the following element is the first element of the output, and the 3 after the blank space is the result.

However in every-day life, where code gets longer, interpreted command lines are often not enough. Therefore, many interfaces dealing with this problem. RStudio is the most common for many of the applications. In standard configuration its user interface is seperated in four quarters. One is the known console, another one lists the variables per environment, the history of commands and connections, the third one displays among others the help page, packages and plots, and last but not least, the scipt window (only if a script is opened). In the script window input can be written in a text file. Lines get executed by pressing Ctrl + Enter (or Ctrl + Return). In order to have R skripts readable later or by others, please mind your coding style [Note: Take some ideas from https://google.github.io/styleguide/Rguide.xml and https://www.r-bloggers.com/\%F0\%9F\%96\%8A-r-coding-style-guide/.]   and use #'_ at the begin of a line to write text, or #.

The # sign symbolises comments in your R code, which will not get executed. It is either used for commenting your code to make it more readable later on or to omit code snippets. For better readability, when commenting for better understanding at the end of the line, leave one two blank spaces to the code and one after #.

#' Section breaks are done with four hash symbols in RStudio (default settings).
#' With the small downwards facing triangle on the left
#####
#### Title
### Subtitle
## Notes from output, e.g. results after time consuming calculations (1.2345678)
# Buzzwords (avoid to use at)
#1 + 2  # This line of code is part of the input, but not interpreted.
#####
#' This is continous running text. 
#' It is used to describe the next part of the code.
#' Pleas also mind the maximum amount of 80 characters per line in such texts, 
#' by manually inserting line breaks.

In this online script output is styled with two #, i. e. ##. You are able to copy the code chunks and run them in your R console.

Blank spaces are not interpreted in R and, thus, they can be placed for readability and optical reasons only.

Be careful, to use parentheses, in order to have the programm code interpreted correctly.

Values can also be stored for further purposes. This can be done in many different ways.

myStorageName <- 1 + 2  # myStorageName gets 1 + 2
#' In other words:
#' 1 + 2 gets assigned to myStorageName. It is stored there.
3 + 4 -> my.storage.name  # Avoid to use.
#' The assignment takes place into the direction of the 'gets-arrow'
assign("mystoragename", 5 + 6)  # assign() ... Assign a value to a name
myStorageName == mystoragename  # == ... (vectorized) comparison for equality
## [1] FALSE

myStorageName <- myStorageName + 3  # the "old" myStorageName woll be overwritten

Object names have to start with a letter.

The assign command can do special assignments in situations where <- does not work as wanted. At the end you can find a glossary with R commands, such as assign.

= can also be used for assign, but is avoided except when handing over parameters to a function. It does not compare two objects.

In contrast, == compares two (vectorised) objects and gives back an object of the sam type. Therefore, it shall be avoided when testing exact equality, to get a single logical value, e. g. in conditions. In such situations identical shall be used.

Getting help in R

How can I get help with built in functions?

The help can be accesed by typing a questionmark and the command without parentheses, i. e. without any argument. If the name of the command is unknown, typing two qustion marks activates a buzzword search. Additionally, when the command does not start with a letter, tick marks or quotation marks have to be used. Executing only the name of the command returns the source code, or – if not coded in R – the head of the function.

?assign
?identical
??'!='  # Click on 'base::<' which is equivalent to the help page of < (?'<')

R is vectorized a.k.a. c is your friend

Ris vectorized. This means that it can calcualte with vectors and matrices without having to break down the operations to simple calculations by hand.

In order to create a vector c( ... ) – for combine – has to be used. The entries of the vector are written inside the parantheses as comma separated values.

c(1, 2, 3)  # a vector with the entries 1, 2 and 3 is generated
## [1] 1 2 3

1:3  # the same result can be achieved creating a sequence with a steplength of 1
## [1] 1 2 3

myVector <- 4:9
myVector^2  # squares all entries of the vector elementwise
## [1] 16 25 36 49 64 81

myVector * myVector  # multiplies the vector elementwise
## [1] 16 25 36 49 64 81

myVector %*% myVector  # multiples the two vectors
##      [,1]  
## [1,]  271

crossprod(myVector)  # built in function for multiplying the two vectors
##      [,1]  
## [1,]  271

myVector2Times <- c(myVector, myVector)  # combine two vectors to one new vector
myVector2Times

In R objects can be generated and saved as well as loaded. Keep in mind, setting your working directory first.

getwd()  # getwd() ... returns the current working directory
## [1] "E:/QuMWiki/Out"

setwd("E:∖∖QuMWiki∖∖Out")  # setwd("SomePath") ... sets the existing working directory
dir.create("E:/QuMWiki/Out")  # dir.create ... creates a new folder

## Warning in dir.create("E:/QuMWiki/Out"): ’E:∖QuMWiki∖Out’ already exists

#' The working directory has to be set additionally ti 'dir.create'
#' The path has to be stated either with "/" or with "∖∖" for Windows.
#' "∖" is an escape character (/ begins something special).
newvec <- c(1, 3, 4, 2, 3, 5)
save(newvec, file="myfirstfile.RData")  # saving the vector 'newvec'
rm("newvec")  # removing the vector 'newvec'
