Definition of matrices

R is not only capable of calculating with vectors, but also capable of calculating with matrices. Thus, in this capter, we will introduce the basics in this chapter.

myMatrix1 <- matrix(1:6, nrow = 2, ncol = 3)  # creates a matrix and assigns it
myMatrix1
##      [,1] [,2] [,3]  
## [1,]    1    3    5  
## [2,]    2    4    6

myMatrix2 <- matrix(1:4, nrow = 2, byrow = TRUE)  # byrow ... fills the matrix rowwise
myMatrix2
##      [,1] [,2]  
## [1,]    1    2  
## [2,]    3    4

str(myMatrix1)  # returns the structure of 'myMatrix1', mind the 'int'
##  int [1:2, 1:3] 1 2 3 4 5 6

class(myMatrix1)  # the object is of class "matrix"
## [1] "matrix"

typeof(myMatrix1)  # the object is of type "integer"
## [1] "integer"

Indexing

To access a matrix, square brackets have to be used, similar to vectors. The only difference is that now two arguments, seperated by a comma, have to be specified.

myMatrix1[1, 1]  # returns the 'top-left' element of the matrix
## [1] 1

myMatrix1[1, 2]  # returns the 'top-second-left' element of the matrix
## [1] 3

myMatrix1[1:2, 1:2]  # returns the 'top-left' 2x2 matrix of "myMatrix1"
##      [,1] [,2]  
## [1,]    1    3  
## [2,]    2    4

myMatrix1[1, ]  # returns the first row of the matrix
## [1] 1 3 5

myMatrix1[, 1]  # returns the first column of the matrix
## [1] 1 2

class(myMatrix1[, 1])  # the object is of class "integer", i.e. is only a vector
## [1] "integer"

R drops dimensions (unless told otherwise)

As seen before, accessing a row or a column leads to a vector. R automatically casts to a simpler class. In some applications it is essential to preserve the character of a row or column vector. This typecasting can be prevented and, consequentely, the class matrix is preserved.

myMatrix1[, 1, drop = FALSE]  # returns the first column of the matrix as column vector
##      [,1]  
## [1,]    1  
## [2,]    2

class(myMatrix1[, 1, drop = FALSE])  # the object still is of class "matrix"
## [1] "matrix"

Indexing via names

Like vectors, also matrices can get names, i. e. rownames and columnnames.

myMatrixRowNames <- c("A", "B", "C", "D")
myMatrixColNames <- c("One", "Two", "Three")
rownames(myMatrix3)  # regarding matrices, the elements are called rownames

colnames(myMatrix3)  # and colnames, repectively
## NULL

rownames(myMatrix3) <- myMatrixRowNames
colnames(myMatrix3) <- myMatrixColNames

## [1] "A" "B" "C" "D"

colnames(myMatrix3)

Market share example

Initially, three firms, A, B, and C, share the market for a certain commodity. Firm A has 20% of the market, B has 60%, and C has the rest. In the course of the next year, the following changes occur: firm A keeps 85% of its customers while losing 5% to B and 10% to C; firm B keeps 55% of its customers, while losing 10% to A and 35% to C; and firm C keeps 85% of its customers, while losing 10% to A and 5% to B. Find the transition matrix T and the market share vector s. Find the values of Ts, T(Ts), TT(Ts), etc.!

–

Firm C has 20%.

The first entry of a column vector is the share of A in the market, the second entry the share of B and the third of C. Thus

s = 0.2 0.6 0.2

When writing the shares for one firm in one row and one state in one column, conequentely

T = 0.85 0.1 0.1 0.05 0.55 0.05 0.1 0.350.85

(transmat <- matrix(c(85, 5, 10, 10, 55, 35, 10, 5, 85), nrow = 3)/100)

To invert matrices solve can be used.

solve(myMatrix2)  # inverts the matrix

Solve is generally used for solving systems of linear equations like this

c1,1x1 + c1,2x2 + c1,3x3 + ⋯ + c1,pxp c2,1x1 + c2,2x2 + c2,3x3 + ⋯ + c2,pxp c3,1x1 + c3,2x2 + c3,3x3 + ⋯ + c3,pxp ⋮ c k,1x1 + ck,2x2 + ck,3x3 + ⋯ + ck,pxp = = = ⋮ = d1 d2 d3 ⋮ d p c1,1c1,2c1,3…c1,p c2,1c2,2c2,3…c1,p c3,1c3,2c3,3…c1,p ⋮ ⋮ ⋮ ⋱ ⋮ c k,1ck,2ck,3…ck,p x1 x2 x3 ⋮ x p = d1 d2 d3 ⋮ d p cx = d

As seen above, if d is the identity matrix, c is inverted.

A simple example of a Leontief model

A simple economy has three industries: fishing, forestry, and boatbuilding. Producing

  • 1 ton of fish requires α = 1 2 fishing boats
  • 1 ton of timber requires β = 1 2 tons of fish
  • 1 fishing boat requires γ = 2 tons of timber

Assume further that this economy has a final demand of d1 = 100 tons of fish and d2 = 80 tons of timber (and no final demand for boats). How many tons of fish x1, how many tons of timber x2, and how many fishing boats x3 have to be produced?

–

The input-output matrix is obtained by writing the final (output) proportions of goods in one row per good. So e. g. fish gets row 1, timber gets row 2 and the category of fishing boats gets row 3.

According to the specifications above of the input-output matrix, the final demand and the system of linear equations, the R-code looks like this:

(iomat <- matrix(c(1, 0, -.5, -.25, 1, 0, 0, -2, 1), nrow = 3))
##      [,1]  [,2] [,3]  
## [1,]  1.0 -0.25    0  
## [2,]  0.0  1.00   -2  
## [3,] -0.5  0.00    1

(demandfinal <- c(100, 80, 0))
## [1] 100  80   0

solve(iomat, demandfinal)
## [1] 160 240  80

apply

Taking the sum of all entries of a vector is done via sum(vector). Taking the sum of all entries of a row/ column of a matrix can be done with the apply command.

myMatrix3  # remember how "myMatrix3" looks
##   One Two Three  
## A   1   5     9  
## B   2   6    10  
## C   3   7    11  
## D   4   8    12

apply(myMatrix3, 1, sum)  # applies the function rowwise (sum of all elements per row)
##  A  B  C  D  
## 15 18 21 24

apply(myMatrix3, 2, sum)  # applies the function columnwise
##   One   Two Three  
##    10    26    42

apply(myMatrix3, 1, max)  # applies the function rowwise (maximum of all elements)
##  A  B  C  D  
##  9 10 11 12
