As you may have noticed, some objects have been used, which were not created initially.

  • Why have I not noticed that at first sight?
  • As you know, creating an Rscript is iteratively done, i. e. a code chunk is written, revised and rewritten.
  • Consequently, I advise adding rm(list=setdiff(ls(), c("These", "variables", "will", "be", "kept"))) at the beginning of your code. This deletes every (none-hidden) object except the listed objects.
  • Additionally, to explicitly perform a garbage collection (give back unused memory to the operating system), gc() can be called.

Thus

rm(list=setdiff(ls(), c("MyFilePath", "MyFilePathOut", "MyRnwFile")))

Profit Maximization under production constraints

What combination of goods x and y should a profit-maximizing firm produce when its total profit function is

Π(x,y) = 80x − 2x2 − xy − 3y2 + 100y

and its maximum output capacity is x + y = 12. Calculate the maximum profits Π as well as the effect on profit if the maximum capacity is expanded by 1 unit.

In this case, a profit function is already given.

As a first step, the profit function has to be transformed into an Rfunction.

profit <- function(quantity) 80 * quantity[1] - 2 * quantity[1]^2 -
  quantity[1] * quantity[2] - 3 * quantity[2]^2 + 100 * quantity[2]

Furthermore, a grid is generated and contourlines are plotted to see the indifference curves of the profit function and the restriction

fineness <- 1000
xmesh <- seq(0, 12, length.out=fineness)
ymesh <- seq(0, 12, length.out=fineness)
mesh <- expand.grid(xmesh, ymesh)
z <- apply(mesh, 1, profit)
str(z)
##  num [1:1000000] 0 0.961 1.921 2.88 3.839 ...

z <- matrix(z, ncol=fineness, byrow=FALSE)
str(z)
##  num [1:1000, 1:1000] 0 0.961 1.921 2.88 3.839 ...

contour(xmesh, ymesh, z=z, nlevels=40)
abline(a = 12, b = -1, col = "red")

The constrOptim command is used to evaluate the quantities of goods, which maximize the profits subject to the given restriction. According to the form of the restriction inequality ( ≥), the u matrix and the c vector are the following:

(matrix.u <- matrix(c(1, 0, 0, 1, -1, -1, 1, 1), 4, 2, byrow = TRUE))
##      [,1] [,2]  
## [1,]    1    0  
## [2,]    0    1  
## [3,]   -1   -1  
## [4,]    1    1

(vector.c <- c(0, 0, -12-1e-5, 12-1e-5))
## [1]   0.00000   0.00000 -12.00001  11.99999

It can be seen that the quantity x of the first good and the quantity y of the second good have to be greater than 0. The explicitly mentioned constraint x + y = 12 is implemented in two ways:

  • − x + (−y) ≥−12.00001

    This equation states that at most 12 units (of a mixture of both goods) can be used ((−1)(−x + (−y)) ≥ (−1)12 ⇔ x + y ≤ 12). Additionally a tolerance is added to the original constraint, respectively subtracted from the transformed constraint, to make it numerically solvable.

  • x + y ≥ 11.99999

    This equation states that at least 12 units (of a mixture of both goods) shall be used (x + y ≥ 12). This constraint is needed due to the fact that our explicitly given constraint targets using all resources. In this special case, the constraint is not needed, but clarifies the ambition of the problem. Additionally a tolerance is subtracted from the original constraint, to make it numerically solvable.

Using the constrOptim function, initial values for our input variables have to be used. Note that those have to be inside the feasible set of possible values.

sol.cOpt=constrOptim(c(6,6), profit, grad = NULL, ui=matrix.u, ci=vector.c,

Linear programming

A firm produces sleeping bags x1 and tents x2. Its profit is 50 dollars per sleeping bag and 30 dollars per tent.

Each sleeping bag requires2 hours for cutting,5 hours for sewing and1 hour for waterproofing.
Each tent requires1 hour for cutting, 5 hours for sewing and3 hours for waterproofing.
The company has at most14 hours for cutting,40 hours for sewing and18 hours for waterproofing a day.

What is the optimal allocation of resources to maximize profit?

Once the constraints are formulated, they are translated into R.

library(lpSolve)
#Deafult is minimisation, for maximisation set direction="max"
#Objective function and constraints should be provided in the vector/matrix form 
(objective <- c(50, 30))
## [1] 50 30

matrix.u.lp <- rbind(
  c(1, 0),
  c(0, 1),
  c(2, 1),
  c(5, 5),
  c(1, 3)
)
directioneq <- c(
  ">=",
  ">=",
  "<=",
  "<=",
  "<="
)
vector.c.lp <- c(
  0,
  0,
  14,
  40,
  18
)

With this input parameters, the lp function can be used.

sol.lp <- lp(
  "max",
  objective.in=objective,
  const.mat=matrix.u.lp, const.dir=directioneq, const.rhs=vector.c.lp
