2.1 QuM2 – Session 1

Profit Maximization

Maximize the profit Π of a firm, given total revenue function

R(Q) = 4000Q − 33Q2

and total cost function

C(Q) = 2Q3 − 3Q2 + 400Q + 5000.

Assume Q > 0.

Be careful to distinguish between the analytical steps for solving this optimization problem and the computational steps for solving this problem in R.

We have to set up the profit function

Π(Q) = −2Q3 − 30Q2 + 3600Q − 5000.

The analytical function has to be transformed into functions in R. For further usage it is saved as a meaningful object and can be used as an ordinary function in R.

revenue<-function(Q) 4000 * Q - 33 * Q^2
cost<-function(Q) 2 * Q^3 - 3 * Q^2 + 400 * Q + 5000
profit <- function (Q) (-2) * Q ^ 3 - 30 * Q^2 + 3600 * Q - 5000

It is often advised to take a quick look at the functions.

curve(revenue, 0, 60, n=100,
      main="Revenues, Costs, Profits",
      xlab="Quantity", ylab=" ",
      lwd=2, col=2)
curve(cost, 0, 60, n=100,
      add=TRUE, lwd=2)
curve(profit, 0, 60, n=100,
      add=TRUE, lwd=2, col=5)
abline(v=20,
       lty=2, lwd=1, col="blue")
legend("topleft", c("Revenue", "Cost", "Profit"),
       lty=1, col=c(2, 1, 5))

The optimize function is used in R for single-variable optimization. Have a look at the entry in the glossary to read about this function.

(Q_star <- optimize(profit, interval=c(0, 100), maximum=TRUE))
## $maximum  
## [1] 20.00001

Due to the numerical calculation, the result is not exactly the expected value of 20. Therefore looking in the help of the optimize command clarifies why:

?optimize

for (tolerance in seq(0.1,0.7,0.1)) {

Profit Maximization, two goods

A firm producing two goods x and y has the profit function:

Π(x,y) = 64x − 2x2 + 4xy − 4y2 + 32y − 14

Find the profit maximizing output for each of the two goods.

In this case, a profit function is already given. It can be plotted using the curve3d command. This command will not be needed, because it needs extra system libraries. In order to solve this example in R, the optim function can be used, although it minimizes the given function. Be careful that the optim command requires a function with one input parameter being a vector.

profit2 <- function(quantities) {
  64 * quantities[1] - 2 * quantities[1]^2 + 4 * quantities[1] * quantities[2] -
    4 * quantities[2]^2 + 32 * quantities[2] - 14
}

The control parameter requires a list containing different parameters. The function value is divided by the fnscale parameter. Thus, the original maximizing problem (highest profits) is turned into a minimizing problem. Consequently, the optim function is a suitable command, as claimed before.

optim(c(0.1, 0.1), profit2, control=list(fnscale=-1))
## $par  
## [1] 40.00004 24.00006

Another possibility to visualize the profit function is as a projection onto the plane spanned by the independent variables. This type of picture is called contour plot. It is achieved in R by using the contour-command.

First a grid is needed, which spans the plane with the independent variables. Next, the function is evaluated at every point forming a matrix. As a last step, the contour command is used.

fineness <- 1000
xmesh <- seq(20, 50, length.out=fineness)
ymesh <- seq(10, 30, length.out=fineness)
mesh <- expand.grid(xmesh, ymesh)
z <- apply(mesh, 1, profit2)
str(z)
##  num [1:1000000] 1186 1187 1187 1188 1189 ...

z <- matrix(z, ncol=fineness, byrow=FALSE)
str(z)
##  num [1:1000, 1:1000] 1186 1187 1187 1188 1189 ...

contour(xmesh, ymesh, z=z, nlevels=15)
